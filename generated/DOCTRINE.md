### ddr_031 — Inspect rendered state before claiming visual correctness

An agent must not infer visible UI correctness from source text alone; it needs direct inspection of the rendered target before explaining or claiming success.

| Source | `rules/process/ddr_031.json` |

#### Why

Source can be correct while runtime composition, state, density, or stale assets remain visibly wrong.

#### Prefer

- Inspect the current rendered target in the relevant state and viewport.
- Report what was observed separately from what source implies.

#### Avoid

- Declaring UI correct because code looks correct.
- Using stale screenshots or an unverified server.

#### Applies When

- An agent explains, reviews, or claims completion of visible UI work.

#### Does Not Apply When

- The task is explicitly source-only and visual state is out of scope, which must be stated.
